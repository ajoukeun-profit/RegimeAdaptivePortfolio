## 1. 어떤 데이터를 받아서 뭘 했나

팀원이 만든 데이터는 이런 구조야:

```
X (입력): 30거래일 × 10개 지표  ← "이 기간의 시장 상황"
y (정답): 0/1/2                ← "5일 후 국면이 Bear/Neutral/Bull"
```

총 698개 샘플. 시간순으로 train(488) / valid(105) / test(105) 분리.

10개 입력 지표는 다음과 같아:

| 지표 | 뜻 |
|------|----|
| log_return | 일별 로그 수익률 |
| return_1d | 일별 단순 수익률 |
| volatility_20d_ann | 20일 변동성 (연율화) |
| ma_gap_20_60 | 20일 이평선 - 60일 이평선 괴리 |
| drawdown_126d | 6개월 최고가 대비 낙폭 |

| rsi_14 | RSI 14일 |
| macd_hist | MACD 히스토그램 |

## 2. 어떤 모델을 만들었나

이름: **Conv1D + LSTM 분류 모델**

쉽게 말하면:

```
[30일치 시장 데이터]
        ↓
[Conv1D]  ← "3~5일 단위 패턴 찾기"  (마치 이동평균처럼)
        ↓
[LSTM]    ← "그 패턴들의 흐름을 시간순으로 기억"
        ↓
[출력]    ← Bear일 확률, Neutral일 확률, Bull일 확률
```

**왜 이 구조인가?**

- Conv1D: 금융 데이터는 하루하루 노이즈가 심하다. 3~5일 단위로 묶어서 패턴을 찾으면 노이즈가 줄어든다.
- LSTM: "3개월 전에 급락했다가 회복한 패턴"처럼 시간 간격이 있는 기억이 필요하다. LSTM은 이런 장기 의존성을 학습할 수 있다.

모델이 출력하는 확률은 이렇게 포트폴리오 비중으로 변환된다:

```
주식 비중 = Bull 확률 + Neutral 확률 × 0.5
현금 비중 = 1 - 주식 비중

예: [p_bear=0.1, p_neutral=0.2, p_bull=0.7]
→ 주식 비중 = 0.7 + 0.2×0.5 = 80%
→ 현금 비중 = 20%
```

## 3. 어떻게 학습시켰나

### 핵심 설계 3가지

**① 클래스 가중치 (Class Weight)**
Neutral 샘플이 Bear, Bull보다 적어서 모델이 Neutral을 무시하는 경향이 있다. 이를 막기 위해 Neutral을 틀리면 더 큰 패널티를 부과했다.

**② Early Stopping**
검증 데이터 성능이 25번 연속 안 올라가면 학습을 멈춘다. 모델이 학습 데이터를 외우는 걸 방지한다.

**③ Learning Rate Scheduler**
학습이 정체될 때 학습률을 절반으로 줄인다. 더 세밀한 수렴을 유도한다.

## 4. 어떤 개선 실험을 했나

총 4가지 실험을 순서대로 진행했다.

### Exp1: 기본 모델 (Baseline)
가장 기본적인 구성. 기준점.

**결과**: 정확도 57.1%, Neutral recall 23.8% (Neutral을 잘 못 맞춤)

---

### Exp2: Focal Loss 적용

**문제**: 기본 모델이 Bull은 97.6% 맞추면서 Neutral은 23.8%밖에 못 맞춘다.

**해결 아이디어**: 이미 잘 맞추는 쉬운 예제(Bull)의 학습 기여를 줄이고, 어려운 예제(Neutral)에 집중하도록 loss 함수를 바꾼다.

```
일반 Loss:  모든 예제를 동등하게 학습
Focal Loss: 틀리기 쉬운 예제를 더 열심히 학습
```

**결과**: Neutral recall 23.8% → **47.6%** (2배 향상!)  
하지만 전체 정확도는 52.4%로 하락. Bull을 희생해서 Neutral을 학습한 것.

---

### Exp3: Focal Loss + 데이터 증강 ← **Phase 1 최고**

**문제**: 학습 데이터가 488개로 너무 적다. 14년치 데이터가 5거래일 간격 라벨링으로 압축되어서.

**해결 아이디어**: 원본 데이터에 아주 작은 랜덤 노이즈를 더한 복사본을 만들어 학습 데이터를 2배로 늘린다.

```
원본 데이터:   [0.5, -0.3, 1.2, ...]
노이즈 추가 후: [0.52, -0.28, 1.19, ...]  ← 거의 같지만 조금 다름
488개 → 976개
```

실제 시장에서도 같은 국면이 매번 조금씩 다른 형태로 나타난다. 노이즈 추가는 이 다양성을 인위적으로 학습시키는 것.

**결과**: 전체 정확도 **61.0%** (4개 실험 중 최고), Bear recall **46.5%** (최고)

---

### Exp4: BiLSTM + Attention

**아이디어**: 기본 LSTM은 day1→day30 한 방향만 읽는다. 양방향(앞→뒤, 뒤→앞)으로 읽으면 더 풍부한 표현을 만들 수 있다. Attention은 "어떤 날짜가 국면 판단에 중요한지" 모델이 스스로 찾게 한다.

**결과**: 정확도 49.5% (최저). 모델이 더 복잡한데 데이터가 너무 적어서 오히려 역효과.

## 5. 최종 분류 성능

| 단계 | 실험 | 전체 정확도 | Bear | Neutral | Bull |
|------|------|---------|------|---------|------|
| Phase 1 | Exp1 Baseline | 57.1% | 34.9% | 23.8% | 97.6% |
| Phase 1 | Exp3 Augmentation | 61.0% | 46.5% | 33.3% | 90.2% |
| Phase 2 | Multi-label (4자산) | 59.8% | 58.8% | 25.3% | 80.6% |
| **Phase 3** | **Cross-asset (최종)** | **61.9%** | **60.5%** | 0.0% | **95.1%** |

참고 기준:
- Random 예측 (3클래스): 33.3%
- 항상 Bull 예측: 39.0%
- **최종 모델: 61.9%** (+28.6%p vs random)

## 6. 포트폴리오 백테스트 결과

모델이 출력한 확률로 비중을 조절하며 2024.04~2026.05 기간을 백테스트했다.

| 전략 | 누적수익 | Sharpe | MDD | Calmar |
|------|--------|--------|-----|--------|
| Buy & Hold | 49.9% | 1.07 | -17.0% | 1.26 |
| 60/40 | 28.3% | 0.84 | -10.4% | 1.22 |
| MA Crossover | 29.1% | 0.82 | -10.9% | 1.19 |
| **Conv1D+LSTM** | **21.9%** | 0.73 | **-7.4%** | **1.35** |

**MDD(최대 낙폭) 최저, Calmar Ratio 최고.**

테스트 기간이 강한 상승장(AI 붐, 미국 금리 인하)이어서 절대 수익률은 낮다. 모델이 방어적으로 행동했기 때문인데, 이는 하락장에서 강점이 된다.

## 7. 중요한 발견

### "분류 정확도가 높다고 투자 성과가 좋지 않다"

Exp3는 분류 정확도 61%로 가장 높지만, 백테스트 누적 수익률은 14.8%로 낮다. Exp3가 Bear를 더 잘 맞추는 만큼 상승장에서도 주식 비중을 줄였기 때문이다.

이것은 금융 ML의 중요한 교훈이다. **포트폴리오 성과와 분류 정확도는 별개의 지표다.**

## 10. 추가 실험: 자산 확장 시도 (음성 결과)

팀원이 논문 기반 ETF 데이터를 추가 제공해줘서 확장 실험을 진행했어.

| 시도 | 결과 | 이유 |
|------|------|------|
| SPY 라벨 2004년부터 재생성 (1003개) | 샘플 증가 성공 | 이전 라벨은 2012년 시작 |
| 15자산 + 1002 샘플로 재학습 | 60.0% (↓1.9%p) | 피처 150개로 과적합 |
| 20자산 + 844 샘플로 재학습 | 55.9% (↓6.0%p) | 피처 200개, 더 심한 과적합 |

**결론**: 자산을 더 넣어도 성능이 안 올랐어. 자산이 많아질수록 입력이 복잡해지는데, 학습 데이터가 그걸 감당할 만큼 충분하지 않아서 오히려 역효과.

최종 모델은 그대로 **Phase 3 (4자산, 61.9%)** 유지.
